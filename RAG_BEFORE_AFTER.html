검색 방식 구조 변화

v0.1 초기: 벡터 검색 단독

쿼리 → 임베딩 → ChromaDB 유사도 검색 → top_k 청크 → LLM
  • 정확한 키워드가 포함된 청크를 놓칠 수 있음
  • 예: "해지 시 환불" 검색 시 "환불"이 직접 언급된 청크 누락 가능

v0.2 완료: 하이브리드 검색

쿼리
 ├─ 임베딩 → 벡터 검색 (의미 유사도)
 └─ 토크나이징 → BM25 검색 (키워드 매칭)
        ↓
    RRF 점수 병합 → top_k 청크 → LLM

측정 가능한 변화: 청크 평균 길이 +100% (실측)

추정 변화: 검색 Recall +5~15% (BEIR 벤치마크 기준)

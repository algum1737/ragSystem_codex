# Security

## Goal

보안 요구사항을 구현 이후가 아니라 설계 단계부터 기록하고 검증 후보로 관리한다.

## Baseline

- 민감 정보는 문서와 코드에서 명시적으로 분리한다.
- 외부 연동에는 최소 권한 원칙을 적용한다.
- 인증, 권한, 비밀 관리 규칙은 후속 설계 문서로 확장한다.

## Current Security Baseline

- 문서 원문과 생성 결과는 로컬 환경에서만 다룬다.
- 외부 LLM API 대신 Ollama 로컬 모델을 사용한다.
- 업로드 파일 크기 상한과 형식 검증은 API 경계에서 유지한다.
- 샘플 문서와 실제 사내 문서는 분리해 관리해야 하며, 이 저장소에는 민감한 실문서를 기본으로 넣지 않는다.

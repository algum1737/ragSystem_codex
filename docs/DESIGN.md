# Design

## Intent

- 제품과 하네스를 함께 설계한다.
- 한 번의 작업 지시가 문서, 구현, 검증으로 이어지게 한다.

## Rules

- 설계 결정은 짧은 문서로 남긴다.
- 큰 기능은 구현 전 설계 요약이 있어야 한다.
- 설계 문서는 문제, 결정, 대안, 리스크를 포함한다.

## Current Focus

- 기존 `ragSystem` 기능을 유지한 채 운영 체계를 Codex 기준으로 전환한다.
- 검색 품질 개선 작업은 항상 평가 파이프라인과 같이 설계한다.
- 새 출력 형식이나 새 도메인 확장은 제품 명세와 실행 계획을 먼저 만든다.

# Exec Plans

- `active/`: 진행 중인 계획
- `completed/`: 완료된 계획

운영 원칙:

- 큰 작업은 구현 전에 먼저 `active/`에 계획을 작성한다.
- 계획 문서에는 최소한 목표, 범위, 가정, 단계, 검증 기준을 남긴다.
- 작업이 완료되면 완료 범위, 남은 작업 여부, 검증 결과를 계획 문서에 기록한다.
- 체크리스트와 검증 결과가 모두 충족되고 추가 `Open Work`가 없으면 `active/`에서 `completed/`로 이동한다.
- 완료 문서로 이동할 때는 이를 참조하는 인덱스, `HANDOFF`, 검증 스크립트도 함께 갱신한다.
- `active/`에 완료된 계획을 방치하지 않는다. 완료 여부가 애매하면 `Open Work`와 다음 판단 기준을 명시한다.

# Frontend

## Intent

- UI는 제품 가치를 전달하는 계층이지만, 도메인 규칙의 저장소가 아니다.
- 에이전트가 읽고 검증하기 쉬운 상태 표현과 UI 구조를 우선한다.

## Rules

- UI 상태와 서버/도메인 상태를 혼합하지 않는다.
- 주요 사용자 흐름은 자동 검증 가능한 시나리오로 남긴다.
- 시각적 변경은 스크린샷 또는 DOM 기반 검증 후보를 함께 기록한다.

## Current Frontend Context

- 현재 프론트엔드는 `app.py` 하나의 Streamlit 앱이다.
- 주요 탭은 인제스천, 쿼리, 시스템, PPT 생성이다.
- UI 변경은 API 계약과 함께 검토해야 하며, 상태 선택값이 문자열 오염 없이 전달되는지 확인해야 한다.
